<task type="auto" tdd="true">
  <name>Task 1: Centralized logger and stuck detector modules</name>
  <files>gsd-runner/src/logger.ts, gsd-runner/src/stuck-detector.ts, gsd-runner/src/types.ts, gsd-runner/src/config.ts, gsd-runner/test/logger.test.ts, gsd-runner/test/stuck-detector.test.ts</files>
  <behavior>
    Logger:
    - logger object exports child loggers: session, telegram, loop, gate, stuck
    - Each child logger includes { component: 'name' } binding
    - Log level configurable via LOG_LEVEL env var (default: 'info')
    - Root logger has name 'gsd-runner'

    StuckDetector:
    - record(toolName, args) returns false when tool calls are diverse
    - record(toolName, args) returns true when same tool+args appears >= threshold times in window
    - Read-only tools (Read, Glob, Grep) require higher threshold (2x) to avoid false positives
    - reset() clears the window
    - Window is bounded (oldest entries evicted when full)
  </behavior>
  <action>
    1. Add to types.ts:
       - `StuckDetectorConfig` interface: `{ windowSize: number; threshold: number; readOnlyMultiplier: number }`
       - Add `logLevel` and `stuckDetector` fields to `RunnerConfig`

    2. Update config.ts to load new env vars:
       - `LOG_LEVEL` (default 'info')
       - `STUCK_WINDOW_SIZE` (default 20)
       - `STUCK_THRESHOLD` (default 5)
       - `STUCK_READONLY_MULTIPLIER` (default 2)

    3. Create logger.ts:
       - Single pino root logger with configurable level and name 'gsd-runner'
       - Export `logger` object with child loggers: session, telegram, loop, gate, stuck
       - Export `createLogger(level?: string)` factory for testing

    4. Create stuck-detector.ts:
       - `StuckDetector` class with constructor(config: StuckDetectorConfig)
       - `record(toolName: string, args: string): boolean` -- hashes tool+args, maintains sliding window, returns true if stuck
       - `reset(): void` -- clears window
       - Read-only tool list: ['Read', 'Glob', 'Grep', 'WebFetch'] -- these get threshold * readOnlyMultiplier
       - Simple djb2 hash for args comparison (not crypto)

    5. Write tests FIRST (TDD):
       - logger.test.ts: verify child logger creation, component bindings, level configuration
       - stuck-detector.test.ts: diverse calls below threshold, identical calls hitting threshold, read-only tool higher threshold, reset clears state, window eviction
  </action>
  <verify>

  </verify>

    - StuckDetector correctly detects stuck agents with configurable thresholds, tests pass

  </done>
