<task type="auto" tdd="true">
  <name>Task 2: Telegram bot module with gate controller, progress, and heartbeat</name>
  <files>gsd-runner/src/telegram.ts, gsd-runner/test/telegram.test.ts</files>
  <behavior>
    Gate Controller:
    - requestGateApproval(message) sends message with inline approve/reject keyboard, returns Promise<boolean>
    - Approve callback resolves Promise with true, removes buttons from message
    - Reject callback resolves Promise with false, removes buttons from message
    - Timeout rejects Promise with Error after gateTimeoutMs
    - answerCallbackQuery() always called (no spinning loader)
    - Catch-all handler answers unmatched callback queries

    Progress:
    - sendProgress(text) sends plain text message to chatId
    - No inline keyboard on progress messages

    Heartbeat:
    - startHeartbeat() begins interval that sends heartbeat messages
    - stopHeartbeat() clears the interval
    - Heartbeat failure logs warning but does not crash

    Lifecycle:
    - start() begins long polling (fire-and-forget, NOT awaited)
    - stop() stops polling and clears heartbeat
  </behavior>
  <action>
    Create telegram.ts as a `TelegramBot` class:

    1. Constructor takes `TelegramConfig` and optional pino logger
    2. Creates `Bot` instance from grammY with config.botToken
    3. Registers callback query handler matching `gate:(approve|reject):<id>` pattern
    4. Registers catch-all callback query handler
    5. Sets up error handler that logs via pino

    Methods:
    - `start(): void` -- calls `bot.start()` WITHOUT await (fire-and-forget). Pass `onStart` callback to log.
    - `stop(): Promise<void>` -- stops heartbeat, calls `bot.stop()`
    - `requestGateApproval(message: string): Promise<boolean>` -- builds InlineKeyboard with Approve/Reject buttons using `gate:approve:<gateId>` / `gate:reject:<gateId>` callback data. Sends message with HTML parse_mode. Returns Promise that resolves on button press or rejects on timeout.
    - `sendProgress(text: string): Promise<void>` -- `bot.api.sendMessage(chatId, text)`
    - `sendAlert(text: string): Promise<void>` -- `bot.api.sendMessage(chatId, '⚠️ ' + text)`
    - `startHeartbeat(): void` -- `setInterval` sending heartbeat message at `heartbeatIntervalMs`
    - `stopHeartbeat(): void` -- `clearInterval`

    Internal state:
    - `pendingGates: Map<number, PendingGate>` for tracking active gate approvals
    - `heartbeatInterval: NodeJS.Timeout | null`

    Write tests FIRST (TDD). Mock the grammY Bot class:
    - Create a minimal mock that captures `sendMessage` calls and simulates callback query handlers
    - Test gate approval flow: send -> callback fires -> Promise resolves true
    - Test gate rejection flow: send -> callback fires -> Promise resolves false
    - Test gate timeout: send -> no callback -> Promise rejects with timeout error
    - Test progress: sendProgress calls sendMessage with plain text
    - Test heartbeat: start/stop manages interval, sends messages
    - Test answerCallbackQuery always called
  </action>
