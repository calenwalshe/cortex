  <behavior>
    - determineNextAction returns { type: 'plan', phase: N } when phase has no plans yet
    - determineNextAction returns { type: 'execute', phase: N } when phase has plans but not all complete
    - determineNextAction returns { type: 'verify', phase: N } when all plans complete but phase not verified
    - determineNextAction returns { type: 'resume' } when .continue-here.md exists
    - determineNextAction returns { type: 'done' } when all roadmap phases are marked [x]
    - determineNextAction returns { type: 'error', reason } when STATE.md is unparseable and no continue-here exists
  </behavior>
  <action>
    1. Create src/state-machine.ts:
       - `determineNextAction(projectDir: string): GsdAction` -- the core pure-ish function
       - Reads `${projectDir}/.planning/STATE.md` and `${projectDir}/.planning/ROADMAP.md` from disk
       - Checks for `${projectDir}/.planning/.continue-here.md` existence (if exists, return resume)
       - Parses both files using state-parser functions
       - Logic (from research Pattern 1):
         a. If .continue-here.md exists -> { type: 'resume' }
         b. Parse state and roadmap
         c. If all roadmap phases complete -> { type: 'done' }
         d. Find current incomplete phase from roadmap
         e. If status contains "Ready to plan" or plansInPhase === 0 -> { type: 'plan', phase: N }
         f. If plansComplete < plansInPhase -> { type: 'execute', phase: N }
         g. If plansComplete === plansInPhase and plansInPhase > 0 -> { type: 'verify', phase: N }
         h. Fallback: { type: 'error', reason: 'Could not determine next action' }
       - Wraps file reads in try/catch. If STATE.md missing or unparseable and no continue-here, returns error action.

    2. Create test/fixtures/continue-here-sample.md with minimal continue-here content.

         - "returns plan action when phase has no plans" (state-ready-to-plan + roadmap-partial)
         - "returns execute action when plans exist but incomplete" (state-executing + roadmap-partial)
         - "returns verify action when all plans complete" (create fixture with plans complete, phase not marked [x])
         - "returns resume when .continue-here.md exists" (any state + continue-here file)
         - "returns done when all phases complete" (any state + roadmap-all-complete)
         - "returns error for missing STATE.md" (empty temp dir)
         - "resume takes priority over other actions" (.continue-here.md present with any state)
