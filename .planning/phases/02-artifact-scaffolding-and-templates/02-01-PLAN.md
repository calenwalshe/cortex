**docs/cortex/clarify/README.md**
- Purpose: Clarify briefs — the problem frame written by `/cortex-clarify`
- Naming: `docs/cortex/clarify/<slug>/<timestamp>-clarify-brief.md`
- Required fields: goal, non-goals, constraints, assumptions, open questions, next research steps
- Created by: `/cortex-clarify <idea>`
- Note: slug is derived from idea text (lowercase, hyphenated); timestamp is ISO 8601

**docs/cortex/research/README.md**
- Purpose: Research dossiers — findings written by `/cortex-research`
- Naming: `docs/cortex/research/<slug>/<phase>-<timestamp>.md`
  - `<phase>` is one of: concept, implementation, evals
- Required sections: summary, findings, trade-offs, recommendations, open questions, sources
- Created by: `/cortex-research --phase <concept|implementation|evals> --depth <quick|standard|deep>`
- Note: each `--phase` invocation produces a separate dossier; phases are not combined

**docs/cortex/specs/README.md**
- Purpose: Specs and GSD handoffs — produced by `/cortex-spec`
- Naming: `docs/cortex/specs/<slug>/spec.md` and `docs/cortex/specs/<slug>/gsd-handoff.md`
- Required fields in spec.md: problem, scope (in/out), architecture decision, interfaces, dependencies, risks, sequencing, tasks, acceptance criteria
- Required fields in gsd-handoff.md: objective, deliverables, requirements, tasks, acceptance criteria, contract link
- Created by: `/cortex-spec` (reads clarify brief + research dossiers from this repo)

**docs/cortex/contracts/README.md**
- Purpose: Execution contracts — the approval gate before any product code is written
- Naming: `docs/cortex/contracts/<slug>/contract-001.md` (repair contracts increment: contract-002.md, etc.)
- Required fields: id, slug, phase, objective, deliverables, scope (in/out), write roots, done criteria, validators, eval_plan, approvals, rollback hints
- Note: `eval_plan` field is mandatory — contracts without it are incomplete (per docs/EVALS.md)
- Created by: `/cortex-spec` (first contract); `/cortex-investigate` (repair contracts)
- Status values: draft | approved | closed

**docs/cortex/evals/README.md**
- Purpose: Eval proposals and plans — written by `/cortex-research --phase evals` and promoted after human approval
- Naming:
  - Proposal: `docs/cortex/evals/<slug>/eval-proposal.md`
  - Plan: `docs/cortex/evals/<slug>/eval-plan.md` (written after human approval of proposal)
- Required fields in eval-proposal.md: dimensions (from 8-candidate matrix), fixtures, rubrics, thresholds, failure taxonomy, approval_required flag
- Required fields in eval-plan.md: approved dimensions, fixtures per dimension, thresholds per dimension, run instructions
- See docs/EVALS.md for the 8-dimension candidate matrix

**docs/cortex/investigations/README.md**
- Purpose: Investigation artifacts — root cause analysis written by `/cortex-investigate`
- Naming: `docs/cortex/investigations/<slug>/` — filename is `<timestamp>-investigation.md`
- Required sections: subject, findings, root cause, evidence, repair recommendations
- May also produce a repair contract at `docs/cortex/contracts/<slug>/contract-NNN.md`
- Created by: `/cortex-investigate [<subject>]`

**docs/cortex/reviews/README.md**
- Purpose: Review artifacts — contract compliance review written by `/cortex-review`
- Naming: `docs/cortex/reviews/<slug>/` — filename is `<timestamp>-review.md`
- Required sections: target, review findings, contract compliance (required — cannot be omitted), recommendations
- Contract compliance section checks active contract's done criteria and validators
- Created by: `/cortex-review [<target>]`

**docs/cortex/audits/README.md**
- Purpose: Security and quality audits written by `/cortex-audit`
- Naming: `docs/cortex/audits/<slug>/` — filename is `<timestamp>-audit.md`
- Required lenses (all 7 must be covered): authentication, data handling, secrets exposure, unsafe tool usage, input validation, dependency risks, misuse vectors
- If a lens is not applicable, an explicit note explaining why must appear in the artifact
- Created by: `/cortex-audit [<target>]`

**docs/cortex/handoffs/README.md**
- Purpose: Continuity files that survive `/clear`, `/compact`, and context exhaustion — the recovery substrate
- Files in this directory (not slug-subdirectories):
  - `current-state.md` — active slug, mode, approval status, recent artifacts, open questions, blockers, next action
  - `open-questions.md` — questions blocking phase transitions
  - `next-prompt.md` — paste-ready restart prompt (3-5 sentences) for use after `/clear`
  - `decisions.md` — log of architectural and design decisions
  - `eval-status.md` — current eval pass/fail state for the active contract
  - `last-compact-summary.md` — written by cortex-postcompact hook after each `/compact` run
- Updated by: `/cortex-status` (manual); cortex session hooks (Phase 4, automated)
- See docs/CONTINUITY.md for field schemas of each file
