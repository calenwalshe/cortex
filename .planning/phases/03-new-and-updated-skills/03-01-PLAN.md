  <action>
Create the directory skills/cortex-clarify/ and write SKILL.md from scratch. This is a net-new skill — no existing file to preserve.

The SKILL.md must follow the standard format observed in all 5 existing skills. Structure:

**Header:** `# Cortex Clarify — Problem Framing`

**User-invocable block:**
- Trigger: `/cortex-clarify`
- Also trigger when: "clarify this idea", "help me frame this", "what problem are we solving", "turn this into a brief"

**Arguments:**
- `/cortex-clarify <idea>` — the idea, problem, or feature as a quoted string or inline text (required)

**Instructions — 5 phases:**

Phase 1: Derive slug
- Slugify the <idea> argument: lowercase everything, replace spaces and non-alphanumeric characters with hyphens, collapse consecutive hyphens to one, strip leading/trailing hyphens
- Example: "add smart retry logic to the API client" → smart-retry-logic
- Record the slug for use in all subsequent steps

Phase 2: Check prerequisite state
- Read .cortex/state.json. If slug field is already set to a different active slug, warn the user and ask to confirm before overwriting the active context.

Phase 3: Populate clarify brief
- Read the template at templates/cortex/clarify-brief.md
- Fill all fields: SLUG (derived above), TIMESTAMP (current time as YYYYMMDDTHHMMSSZ), STATUS (draft), IDEA (verbatim input), GOAL (one sentence outcome), NON_GOALS (explicit exclusions), CONSTRAINTS (hard limits), ASSUMPTIONS (things assumed true), OPEN_QUESTIONS (actionable questions that must be answered before research begins), NEXT_RESEARCH_STEPS (ordered agenda for /cortex-research --phase concept)
- Think carefully about each section — ask the user clarifying questions if the idea is too sparse to derive non-goals, constraints, or open questions. Do not silently leave fields empty.

Phase 4: Write artifact
- Target path: docs/cortex/clarify/{slug}/{timestamp}-clarify-brief.md
- Create the directory if it does not exist: mkdir -p docs/cortex/clarify/{slug}/
- Write the populated template to the target path

Phase 5: Update continuity state
- Update docs/cortex/handoffs/current-state.md:
  - slug: {derived slug}
  - mode: clarify
  - approval_status: pending
  - active_contract_path: (none)
  - recent_artifacts: append docs/cortex/clarify/{slug}/{timestamp}-clarify-brief.md
  - open_questions: list from brief
  - blockers: (none unless discovered)
  - next_action: Run /cortex-research --phase concept to begin concept research
- Update .cortex/state.json:
  - slug: {derived slug}
  - mode: clarify
  - approval_status: pending
  - active_contract: null
  - Append artifact path to artifacts array
  - gates.clarify_complete: true
  - gates.research_complete: false (unchanged)
  - gates.spec_complete: false (unchanged)

**Rules (must be explicit in SKILL.md):**
- Does not start research or spec — clarify brief is a prerequisite artifact only
- Does not modify GSD planning state (.planning/, STATE.md)
- The clarify brief is the required gate to /cortex-research; research cannot begin without one
- Output is always written as a repo-local artifact — chat-only responses do not satisfy this command

**Output format block:** After completing all phases, output a terminal summary:
```

════════════════════════════════════════

  - {question 1}
  - {question 2}

```
