**Header:** `# Cortex Spec — Intelligence-to-Execution Handoff`

**User-invocable block:**
- Trigger: `/cortex-spec`
- Also trigger when: "write the spec", "create a spec", "generate handoff", "spec this out", "create the contract"

**Arguments:**
- `/cortex-spec` — no flags or arguments; always operates on the current active slug from .cortex/state.json

**Instructions — 5 phases:**

Phase 1: Validate prerequisites
- Read .cortex/state.json to get the active slug
- Check docs/cortex/clarify/{slug}/ — if no clarify brief exists, block: "No clarify brief found for slug '{slug}'. Run /cortex-clarify first."
- Check docs/cortex/research/{slug}/ — if no research dossier exists, block: "No research dossier found for slug '{slug}'. Run /cortex-research --phase concept first."
- Read all available research dossiers for the active slug (all files matching docs/cortex/research/{slug}/*.md)
- Read the clarify brief

Phase 2: Synthesize spec
- Read template at templates/cortex/spec.md
- Populate ALL 9 mandatory sections — omitting any section is an error:
  1. Problem: what is being built and why (one paragraph, describes the problem not the solution)
  2. Scope: in-scope items and explicit out-of-scope exclusions
  3. Architecture Decision: chosen approach, rationale, alternatives considered and rejected
  4. Interfaces: external interfaces touched (APIs, contracts, module boundaries, file paths; what this spec reads vs writes)
  5. Dependencies: libraries, services, or other Cortex artifacts this spec depends on
  6. Risks: list of risks with mitigation per risk
  7. Sequencing: ordered implementation steps, each producing a verifiable checkpoint
  8. Tasks: discrete implementation tasks as checkbox items, small enough to commit atomically
  9. Acceptance Criteria: measurable, testable criteria with clear pass/fail definitions
- Write to: docs/cortex/specs/{slug}/spec.md
- Create directory: mkdir -p docs/cortex/specs/{slug}/

Phase 3: Write GSD handoff
- Read template at templates/cortex/gsd-handoff.md
- Populate from the synthesized spec — this is the GSD-ready work order for explicit human import
- Write to: docs/cortex/specs/{slug}/gsd-handoff.md

Phase 4: Write first execution contract
- Read template at templates/cortex/contract.md
- Populate all required fields: id (generate as {slug}-001), slug, phase (execute), objective, deliverables (from spec tasks), scope (from spec scope), write_roots (from spec interfaces), done_criteria (from spec acceptance criteria), validators, approvals, rollback_hints
- The eval_plan field is mandatory — include a placeholder path docs/cortex/evals/{slug}/eval-plan.md if no eval plan exists yet; contracts without this field are incomplete
- Contract numbering: contract-001.md (subsequent repair contracts increment the counter)
- Write to: docs/cortex/contracts/{slug}/contract-001.md
- Create directory: mkdir -p docs/cortex/contracts/{slug}/

Phase 5: Update continuity state
- Update docs/cortex/handoffs/current-state.md:
  - mode: spec
  - approval_status: pending (spec and contract require human approval before execution)
  - active_contract_path: docs/cortex/contracts/{slug}/contract-001.md
  - recent_artifacts: append spec.md, gsd-handoff.md, contract-001.md paths
  - next_action: Human must review and approve spec.md and contract-001.md before execution. Import gsd-handoff.md into GSD explicitly — do NOT run GSD commands from this skill.
- Update .cortex/state.json:
  - mode: spec
  - approval_status: pending
  - active_contract: docs/cortex/contracts/{slug}/contract-001.md
  - Append all three artifact paths to artifacts array
  - gates.spec_complete: true

**Rules section (must be explicit in SKILL.md):**
- Requires clarify brief AND at least one research dossier — blocks if either missing
- Does NOT auto-invoke GSD; the human must explicitly import gsd-handoff.md into GSD as a separate step
- The spec and contract require human approval before any execution begins — this is a hard gate
- All 9 spec sections are mandatory; omitting any section is an error

**Output format block:**
```
SPEC WRITTEN
════════════════════════════════════════
Slug:     {slug}
Spec:     docs/cortex/specs/{slug}/spec.md
Handoff:  docs/cortex/specs/{slug}/gsd-handoff.md
Contract: docs/cortex/contracts/{slug}/contract-001.md

Status: PENDING HUMAN APPROVAL

Next steps:
  1. Review docs/cortex/specs/{slug}/spec.md
  2. Review docs/cortex/contracts/{slug}/contract-001.md
  3. Approve both (update Status: draft → approved)
  4. Import gsd-handoff.md into GSD manually
     (do not run GSD commands from this skill)
════════════════════════════════════════
```
